Predicting Car Crash Injury Severity Using Naive Bayes Classification

In this tutorial we will be predicting the injury level of car crashes based on data provided by police. We wil use two datasets from Montgomery County, Maryland. The data is collected via the Automated Crash Reporting System (ACRS). Specifically, we will use two datasets:

  1. Incidents Report: This dataset provides general information on the crash.
  2. Drivers Report: This dataset provides information on the drivers involved in the crash, including injury.

This data is useful for understand the factors that can cause the most harm in a car crash. By building our classifier we will be able to correlate each variable's values with injury severity. This gives us a rough estimate into what plays into the general severity of a car crash. These results could be used to help design roads to be safer, or implement new lessons into driver's education.

The data can be gathered through an HTTP API or by loading CSVs. For simplicity's sake I've downloaded the CSV data.

We'll load the data into a pandas dataframe using the read_csv function. Using a dataframe will allow us to quickly wrangle the data, and perform effecient batch operations. Although using a plain python dictionary or array may be easier initial, it would be detrimental for performance in the long run. Data Ethics

When working with any dataset its important to understand where the data comes from. Who collected it and how it was collected are important in understanding the implications of our results.

Bad data can lead to biased results. When we base our models on prior statistics, the underlying biases in those statistics will show in the models. For example, when Amazon's AI recruiting model tended to prefer men, because most existing engineers at Amazon were men.

With this in mind, we should take a more detailed look at our data source before diving into classification. On the Montgomery County website they list the following disclaimers about the data:

  • Information not yet verified by further investigation
  • Information that may include verified and unverified collision data
  • Preliminary collision classifications may be changed at a later date based upon further investigation
  • Information may include mechanical or human error

Particularly, this means two things for our model:

  1. Injury data may not be accurate. Except in extreme cases the actual severity of an injury may not be obvious at the crash site.
  2. This data is entered by police officers. Fields like "Driver Distracted By", "Road Condition" and can be subjective, or include subtleties not immediately obvious to the police officer at the scene.

Analysis

Now, with our features selected we can implement the classifier.

The core of a Naive Bayes classifier is a table of probabilities for each value of every feature given a specific event. In our case the event is the injury severity.

To apply the model we select the probability associated with each feature's value in the test data set, then aggregate them to determine the likelyhood of each event.

Implementing the Classifier

In the previous section we look at the probability of "Vehicle Impact Location" being "TWELVE OCLOCK" when injury severity was "NO APPARENT INJURY" versus when it was not. This basic idea is the core of multinomial naive bayes classification algorithm.

We want to know the probability of a given feature's value in the event of "NO APPARENT INJURY", "POSSIBLE INJURY", etc. so that we can apply these probabilities to our testing set.

In[10]:

# We'll use this dataclass to store information about the training sets, usefull when running the model

    counts: pd.DataFrame

    categ_probabilities: pd.Series

def train(training_data):
    # this dictionary will map feature name -> feature value count
    counts_dict = {}

    # we want the value counts of each feature's values within given a specific injury severity
    groups = training_data.groupby("Injury Severity")
    for c in FEATURES:
        # get the value counts
        counts_dict[c] = groups[c].value_counts(dropna=False).unstack(fill_value=0)

        # if a specific value wasn't in training set, but is in the dataset set its count to zero
        for v in ALL[c].unique():
            if v not in counts_dict[c].columns:
                counts_dict[c][v] = 0

    # create a multi-level data frame from the counts.
    counts = pd.concat(counts_dict, axis=1)

    # calculate the probability of each value appearing at the given injury level
    # notice we add 1 to both sides of the fraction.
    # This is called "Laplace Smoothing", it eliminates zero probabilities in the result.
    probabilities = counts.apply(lambda col: (col + 1) / (col.sum() + 1), axis="columns")

    # calculate the chance of any giving record falling it a specific injury severity 
    categ_probabilities = training_data["Injury Severity"].value_counts() / training_data.size
    categ_probabilities.index.name = "Injury Severity"
    return CrashModel(
        counts = counts,
        probabilities = probabilities,
        categ_probabilities = categ_probabilities)

def select_probabilities(test_feature_column, model):
    """Select the probabilities for each value of a single feature from the test set, using model"""
    def get_probabilities(injury_categ_row):
        # given a specific injury category row, select the given feature values from it for test_feature_column
        # then convert the result to a dataframe, and reindex using the original test feature column.
        # changing the index will allow us to merge these probabilities back to the original dataframe later
        return injury_categ_row.loc[test_feature_column.name][test_feature_column.values].to_frame(injury_categ_row.name).set_index(test_feature_column.index)

    # x is now a series of dataframes
    x = model.probabilities.apply(lambda p: get_probabilities(p), axis="columns")
    x = pd.concat(x.values, axis="columns")
    x.name = test_feature_column.name
    return x

def predict(test_set, model):
    # for each feature, select the probabilities from the model
    # join the result into a multi-level dataframe that maps (ID, Injury Severity) -> Feature Probabilities
    results = test_set[list(FEATURES)].apply(lambda c: select_probabilities(c, model).stack(), axis="index")
    results.index.names = ("ID", "Injury Severity")
    # get the total probability for each injury severity level for every row, then multiply by the total chance of that row being within a particular severity.
    prods = results.prod(axis="columns").mul(model.categ_probabilities)

    # Identify the most probable injury severity level, then merge those back into the training set
    predictions = prods.groupby(level=0).idxmax().map(lambda x: x[1]).rename("Injury Severity Predicted")
    return test_set.join(predictions)
